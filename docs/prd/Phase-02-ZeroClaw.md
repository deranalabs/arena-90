# Arena90 — Phase 02: ZeroClaw Core Logic

**Status:** Approved
**Author:** Nagi (Hermes Agent) — Revised for ZeroClaw Runtime
**Date:** 2026-06-27

## 1. Goal
Configure the ZeroClaw agent runtime to host ISAGI and AIKU, and build the custom tool (Python) they will use to read the TxLINE mock data.

## 2. Scope
- ZeroClaw configuration (`zeroclaw.toml` or equivalent YAML) defining two agent aliases: `isagi` and `aiku`, including their distinct System Prompts (Risk profiles & playstyles).
- A lightweight Python script (registered as a ZeroClaw tool) to parse `/backend/solana-actions/mock/txodds-mock.json`.
- Output generation: A script that runs both agents via the ZeroClaw CLI and saves their deterministic clash to `/backend/solana-actions/mock/clash-state.json` for the Express backend to serve.

## 3. User Stories
- **US-2.1:** As the Arena orchestrator, I want ISAGI (ZeroClaw Agent A) to read the odds and output an "Over 2.5" prediction if it detects attacking momentum.
- **US-2.2:** As the Arena orchestrator, I want AIKU (ZeroClaw Agent B) to read the same odds and output an "Under 2.5" prediction if it calculates high defensive probability.
- **US-2.3:** As the Node.js backend, I want to read `clash-state.json` (generated by the clash runner) so I know which match to broadcast to Twitter via Blinks.

## 4. Functional Requirements
- **FR-1:** Create `/agents/zeroclaw/zeroclaw.toml` defining agents `isagi` and `aiku`. Set their system prompts to embody their soccer/tactical personas.
- **FR-2:** Create `/agents/zeroclaw/tools/read_odds.py` to parse the mock TxLINE data.
- **FR-3:** Create `/agents/zeroclaw/run_clash.sh` (or a Python runner) that executes `zeroclaw agent -a isagi ...` and `zeroclaw agent -a aiku ...`, extracts their outputs, and writes to `clash-state.json`.

## 5. Definition of Done
- `zeroclaw.toml` is syntactically valid.
- Executing `run_clash.sh` successfully leverages the ZeroClaw CLI to generate a valid `clash-state.json`.
